# FAQ

## よくある質問

### 基本

**zERC20 とは何ですか？**

zERC20 は、Ethereum および EVM 互換チェーン上でプライベート転送（Private Transfer）を実現する ERC-20 準拠のトークンです。通常のトークンでは全ての転送が公開されますが、zERC20 は ZKP（Zero-Knowledge Proof）を使って送信者と受信者の紐付けを隠します。

**zERC20 と Tornado Cash の違いは何ですか？**

どちらもプライバシーのために ZKP を使っていますが、主な違いは以下の通りです：

| 特徴      | zERC20                         | Tornado Cash       |
| ------- | ------------------------------ | ------------------ |
| トークン種別  | ERC-20 ラッパートークン（zUSDC、zETH など） | ETH / ERC-20 デポジット |
| 固定金額    | なし — 任意の金額                     | あり — 固定プールサイズ      |
| クロスチェーン | あり — LayerZero 経由              | なし                 |
| ウォレット操作 | 標準的な MetaMask 送金               | 専用の入出金 UI が必要      |

**どのトークンをサポートしていますか？**

現在サポートされているラッパートークン（Wrapper Token）：

* zUSDC（USDC のラッパー）
* zETH（ETH のラッパー）
* zBNB（BNB のラッパー）

サポートされているチェーンの一覧はフロントエンドで確認してください。

### プライバシー

**転送金額は誰かに見えますか？**

はい、転送・引き出しの金額はオンチェーン上で公開されます。金額プライバシーを保護するには：

* バッチ引き出し（Batch Withdrawal）を使って複数の転送をまとめる
* 一部引き出し（Partial Withdrawal）を使って正確な金額の紐付けを防ぐ
* 一般的な端数のない金額を使う

**送信者に私の引き出し先アドレスは見えますか？**

バーンアドレス（Burn Address）を生成したのが誰かによって異なります：

* **送信者が生成した場合**：はい、送信者はあなたの引き出し先アドレスを知っています（フロントエンドはこの方式のみ対応）
* **受信者が生成した場合**：いいえ、送信者にはバーンアドレスしか見えません（CLI のみ対応）

**IPアドレスは漏れますか？**

zERC20 自体は IP アドレスを公開しませんが：

* トランザクション送信時、RPCプロバイダーはあなたの IP を確認できます
* 追加の IP プライバシーが必要な場合は VPN や Tor の使用をご検討ください

### 手数料とリワード

**Unwrap で手数料がかかるのはなぜですか？**

流動性が低いチェーンで Unwrap する場合に手数料が発生します。手数料はチェーン間の流動性バランスを保つためのインセンティブです。手数料が高い場合は、クロスチェーンUnwrap を使って手数料の低い別チェーンの流動性にアクセスすることをご検討ください。

**高い Unwrap 手数料を避けるには？**

**クロスチェーンUnwrap** を使って、流動性の多い別チェーン経由で Unwrap する方法があります。フロントエンドで全オプションの手数料比較が確認できます。

**Wrap 時にボーナストークンを受け取ったのはなぜですか？**

チェーンの流動性が低い場合、Wrap で流動性を追加するとリワードを獲得できます。このリワードは、過去の Unwrap で蓄積された手数料サープラス（Fee Surplus）から支払われます。

**現在の手数料を確認するには？**

フロントエンドで Wrap / Unwrap モーダルを開き、金額を入力してください。確認前に予想される手数料またはリワードが表示されます。

詳細は [手数料とリワード](fees-and-rewards.md) を参照してください。

### 転送

**プライベート転送はどのくらい時間がかかりますか？**

| ネットワーク  | 目安の時間                           |
| ------- | ------------------------------- |
| Mainnet | 30分〜1時間                         |
| Testnet | LayerZero の不安定性によりさらに時間がかかる場合あり |

**転送をキャンセルできますか？**

バーンアドレスに送金した zERC20 は、回収・リダイレクトが一切できません。引き出せるのは、そのバーンアドレスの正規の受信者のみです（ZKP を使って引き出し権限を証明します）。

**間違ったバーンアドレスに送金してしまいました**

そのバーンアドレスの正規の受信者が引き出しを行わない限り、トークンは永久にロックされます。送金前にバーンアドレスを必ず確認してください。

## トラブルシューティング

### プライベート転送が届いていない

**症状：** バーンアドレスに zERC20 を送金したが、受信者に届いていない。

**対処法：**

1. **クロスチェーンメッセージの伝播を待つ**
   * クロスチェーン転送は、LayerZero メッセージの伝播に30分〜1時間かかります
   * Testnet ではネットワークの不安定性によりさらに時間がかかる場合があります
2. **トランザクションのステータスを確認する**
   * 送信元チェーンで元のトランザクションが確認済みかチェックする
   * ブロックエクスプローラーでバーンアドレスの残高を確認する
3. **Invoice のステータスを確認する**
   * フロントエンド：ページを更新する
   * CLI：`zerc20-cli invoice status --chain-id <CHAIN_ID> --invoice-id <INVOICE_ID>` を実行する
4. **バーンアドレスを確認する**
   * 送金者が正しいバーンアドレスを使用しているか確認する
   * EVMアドレスは大文字・小文字を区別しません（例：`0xAbc...` と `0xabc...` は同じアドレス）が、タイポ防止のためチェックサム形式（大小混在・EIP-55）の使用を推奨します

### トークンがバーンアドレスで止まっている

**症状：** zERC20 がバーンアドレスに表示されているが、引き出せない。

**これは正常な動作です。** バーンアドレスに送られたトークンは永続的にそこに残ります — 引き出し時にトークンが「移動」するわけではありません。代わりに、ZKP の検証によって受信者のアドレスに同額がミントされます。

「残留している」トークンは、ミントされた引き出しを裏付けるバーン済み供給量を示しています。
